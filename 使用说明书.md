# 元数据管理查询接口 - 使用说明书

> 本文档面向接口调用方，提供完整的调用示例、场景说明和常见问题解答。

---

## 一、接口概述

本套 API 提供元数据的统一查询能力，包括：

- **逻辑模型查询**：查询逻辑模型列表、详情、物理模型映射、列定义
- **血缘关系查询**：查询逻辑模型的上下游血缘链路
- **数据质量查询**：查询数据质量核查报告及详情
- **TMF SID 目录查询**：查询 TMF SID 目录树结构，以及目录下关联的逻辑模型

**Base URL：**

| 环境 | URL |
|------|-----|
| 正式环境 | `https://api.example.com/api/v1` |
| 测试环境 | `https://api-test.example.com/api/v1` |

## 二、认证说明

所有接口调用前需要获取 Access Token，并通过 `Authorization` Header 传递。

### 2.1 获取 Token

请联系系统管理员获取 Access Token。

### 2.2 调用时携带 Token

```bash
-H 'Authorization: Bearer your_access_token_here'
```

### 2.3 示例

```bash
curl -k -X GET 'https://api.example.com/api/v1/logical-models?tenantId=example_tenant' \
  -H 'Authorization: Bearer 6eab8b...d7dd'
```

## 三、通用说明

### 3.1 统一响应格式

所有接口均返回相同格式：

```json
{
  "code": "success",
  "msg": "Query success",
  "data": { ... }
}
```

- `code=success` 表示成功
- `code=fail` 表示失败，`msg` 中包含错误原因

### 3.2 分页参数

列表查询接口支持 `pageNum` 和 `pageSize` 参数：

```bash
# 获取第2页，每页50条
curl ... 'https://api.example.com/api/v1/logical-models?tenantId=xxx&pageNum=2&pageSize=50'
```

### 3.3 时间格式

- 日期：`YYYY-MM-DD`（如 `2024-04-01`）
- 时间戳：Unix 毫秒（如 `1713446400000`）
- 日期时间字符串：`YYYY-MM-DD HH:mm:ss`

**返回示例（关键字段）：**

```json
  "msg": "Query success",
  "data": {
    "list": [
      {
        "tableName": "dim_user",
        "labelCh": "用户维度表",
        "labelEn": "User Dimension",
        "status": "release",
        "openFlag": 1,
        "appName": "datacompass"
      },
      {
        "tableName": "xdr_user_behavior",
        "labelCh": "用户行为明细表",
        "labelEn": "User Behavior Detail",
        "status": "release",
        "openFlag": 1,
        "appName": "datacompass"
      }
    ],
    "pagination": {
      "pageNum": 1,
      "pageSize": 500,
      "total": 1200,
      "pages": 3
    }
  }
}
```

**分页遍历所有页：**

```python
import requests

base_url = "https://api.example.com/api/v1"
tenant_id = "example_tenant"
token = "your_token_here"

headers = {"Authorization": f"Bearer {token}"}

all_models = []
page_num = 1

while True:
    params = {"tenantId": tenant_id, "status": "release", "pageNum": page_num, "pageSize": 500}
    resp = requests.get(f"{base_url}/logical-models", headers=headers, params=params)
    data = resp.json()

        print(f"Error: {data['msg']}")
        break
    
    page_data = data["data"]["list"]
    all_models.extend(page_data)
    
    pagination = data["data"]["pagination"]
    if page_num >= pagination["pages"]:
        break
    
    page_num += 1

print(f"Total models fetched: {len(all_models)}")
```

## 六、场景三：查询逻辑模型血缘关系

**需求：** 查询 `dim_user` 的上下游血缘链路，用于数据 lineage 图谱展示。

**调用接口：** `GET /logical-models/{tableName}/lineage`

**请求示例：**

```bash
curl -k -X GET 'https://api.example.com/api/v1/logical-models/dim_user/lineage?tenantId=example_tenant' \
  -H 'Authorization: Bearer <your_token>'
```

**返回示例：**

```json
{
  "code": "success",
  "msg": "Query success",
  "data": {
    "logicalModelName": "dim_user",
    "logicalModelLabelCh": "用户维度表",
    "forwardLineage": [
      {
        "logicalModelName": "ods_user_raw",
        "logicalModelLabelCh": "用户原始数据",
        "forwardLineage": [],
        "backwardImpact": []
      }
    ],
    "backwardImpact": [
      {
        "logicalModelName": "rpt_user_stats",
        "logicalModelLabelCh": "用户统计报表",
        "forwardLineage": [],
        "backwardImpact": []
      }
    ]
  }
}
```

**血缘结构解读：**

```
ods_user_raw  →  dim_user  →  rpt_user_stats
(前向血缘)      (当前模型)    (后向影响)
```

**血缘拉平处理示例（用于写入 Datahub Lineage）：**

```python
def extract_lineage_edges(lineage_node, edges=None):
    if edges is None:
        edges = []
    
    current_name = lineage_node["logicalModelName"]
    
    # 前向血缘 → 当前节点依赖上游
    for upstream in lineage_node.get("forwardLineage", []):
        edges.append({
            "source": upstream["logicalModelName"],
            "target": current_name,
            "type": "derives_from"
        })
        extract_lineage_edges(upstream, edges)
    
    # 后向影响 → 下游依赖当前节点
    for downstream in lineage_node.get("backwardImpact", []):
        edges.append({
            "source": current_name,
            "target": downstream["logicalModelName"],
            "type": "influences"
        })
        extract_lineage_edges(downstream, edges)
    
    return edges

# 调用
lineage = get_lineage("dim_user", "example_tenant", "your_token")
edges = extract_lineage_edges(lineage)
print(f"Total lineage edges: {len(edges)}")
for edge in edges:
    print(f"  {edge['source']} --[{edge['type']}]--> {edge['target']}")
```

```python
import requests

def get_low_score_reports(tenant_id, token, threshold=90):
    headers = {"Authorization": f"Bearer {token}"}
    params = {
        "tenantId": tenant_id,
        "pageNum": 1,
        "pageSize": 500
    }
    
    all_reports = []
    page_num = 1
    
    while True:
        params["pageNum"] = page_num
        resp = requests.get(
            "https://api.example.com/api/v1/data-quality/reports",
            headers=headers,
            params=params
        )
        data = resp.json()
        
        if data["code"] != "success":
            break
        
        page_reports = data["data"]["list"]
        # 筛选低分报告
        low_score = [r for r in page_reports if r.get("score", 0) < threshold]
        all_reports.extend(low_score)
        
        pagination = data["data"]["pagination"]
        if page_num >= pagination["pages"]:
            break
        page_num += 1
    
    return all_reports

# 调用
low_reports = get_low_score_reports("example_tenant", "your_token", threshold=90)
print(f"Found {len(low_reports)} low score reports")
for r in low_reports:
    print(f"  {r['jobInsName']}: score={r['score']}")
```

```python
import requests

def analyze_dq_report(job_ins_id, token):
    headers = {"Authorization": f"Bearer {token}"}
    url = f"https://api.example.com/api/v1/data-quality/reports/{job_ins_id}"
    
    resp = requests.get(url, headers=headers)
    data = resp.json()
    
    if data["code"] != "success":
        raise ValueError(f"Failed: {data['msg']}")
    
    report = data["data"]
    print(f"Report: {report['jobInsName']}")
    print(f"Score: {report['score']}")
    print(f"Total Rows: {report['sumLineNum']}")
    
    # 分析未通过的规则
    failed_rules = []
    for detail_list in [
        report.get("jobInstanceDetailTableList", []),
        report.get("jobInstanceDetailSingTableList", []),
        report.get("jobInstanceDetailFieldList", [])
    ]:
        for rule in detail_list:
            if rule.get("isCheckPass") == 0:
                failed_rules.append({
                    "ruleCode": rule.get("ruleCode"),
                    "fields": rule.get("fields"),
                    "badRecord": rule.get("badRecord"),
                    "totalRecord": rule.get("totalRecord")
```

```bash
# 再查询 ABE 层

  -H 'Authorization: Bearer <your_token>'
```

**返回示例（Domain 层）：**

```json
{
  "code": "success",
  "msg": "Query success",
  "data": [
    {
      "catalogId": "D001",
      "catalogNameCn": "客户域",
      "catalogNameEn": "Customer Domain",
      "catalogLevel": "1",
      "children": [
        {
          "catalogId": "ABE001",
          "catalogNameCn": "客户账户ABE",
          "catalogLevel": "2",
          "children": []
        }
```

**递归构建目录树示例：**

```python
import requests

def build_catalog_tree(family_catalog_type_id, token, data_type_ids=None):
    """
    构建完整目录树
    data_type_ids: [1, 2, 3, 4] 对应 Domain/ABE/BE/业务对象
    """
    if data_type_ids is None:
        data_type_ids = ["1", "2", "3", "4"]
    
    headers = {"Authorization": f"Bearer {token}"}
    
    catalog_map = {}  # catalogId -> node
    
    for dt in data_type_ids:
        params = {
            "familyCatalogTypeId": family_catalog_type_id,
            "dataTypeId": dt
        }
        resp = requests.get(
            "https://api.example.com/api/v1/tmf-sid-catalogs",
            headers=headers,
            params=params
        )
        data = resp.json()
        
        if data["code"] != "success":
            continue
        
        for node in data["data"]:
            catalog_id = node["catalogId"]
            node["dataTypeId"] = dt
            node["children"] = node.get("children", [])
            catalog_map[catalog_id] = node
    
    # 构建父子关系
    roots = []
    for node in catalog_map.values():
        parent_id = node.get("parentCatalogId")
        if parent_id and parent_id in catalog_map:
            catalog_map[parent_id]["children"].append(node)
        else:
            roots.append(node)
    
    return roots

tree = build_catalog_tree("FC001", "your_token", ["1", "2", "3", "4"])
print(f"Root catalogs: {len(tree)}")
```

### 11.1 对接架构

```
你的系统（已有接口）
        ↓
```

### 11.2 适配层开发建议

**定时全量同步：**

```python
import schedule
import time
import requests

def sync_all_logical_models():
    """每日全量同步所有逻辑模型到 DataHub"""
    page_num = 1
    total_synced = 0
    
    while True:
        params = {
            "tenantId": "your_tenant",
            "status": "release",
            "pageNum": page_num,
            "pageSize": 500
        }
        resp = requests.get(
            "https://api.example.com/api/v1/logical-models",
            headers={"Authorization": "Bearer your_token"},
            params=params
        )
        data = resp.json()
        
        if data["code"] != "success":
            break
        
        page_data = data["data"]["list"]
        for model in page_data:
            # 转换成 DataHub Dataset snapshot
            # 调用 DataHub API 写入
            write_to_datahub(model)
            total_synced += 1
        
        pagination = data["data"]["pagination"]
        if page_num >= pagination["pages"]:
            break
        page_num += 1
    
    print(f"Synced {total_synced} logical models to DataHub")

# 每日凌晨2点执行
schedule.every().day.at("02:00").do(sync_all_logical_models)

while True:
    schedule.run_pending()
    time.sleep(60)
```

**关键映射规则：**

| 本 API 字段 | DataHub Aspect | 说明 |
|------------|---------------|------|
| tableName | dataset name | DataHub Dataset 名称 |
| logicalColumns | schema | Dataset schema 中的字段 |
| relatedPhysicalModels | dataPlatformInstance | 物理存储信息 |
| logicTableRels | entityLineage | 血缘边 |

### 11.3 注意事项

1. **Token 管理**：建议将 Token 存储在配置中心或环境变量，不要硬编码
2. **分页限制**：每次最多拉取 500 条，大数据量时分页遍历
3. **错误重试**：接口调用失败时建议做 3 次重试，间隔递增（1s, 5s, 30s）
4. **时间同步**：使用时间戳时注意时区，统一用 UTC 或指定时区

## 十二、常见问题

### Q1: 返回 `code=fail`，`msg=Invalid param`？

**原因：** 必选参数未传或格式错误。

**排查：**
1. 检查 `tenantId` 是否传入
2. 检查 `dataTypeId` 是否在 `["1","2","3","4","10"]` 范围内（注意是字符串，不是数字）
3. 检查 `pageNum`/`pageSize` 是否为正整数

### Q2: 逻辑模型详情里 `relatedPhysicalModels` 是空的？

**原因：** 该逻辑模型尚未关联物理表，属于设计阶段尚未落地的模型。

### Q3: 血缘接口返回的 `forwardLineage` 和 `backwardImpact` 都是空数组？

**原因：** 该逻辑模型的血缘关系尚未录入，或血缘录入方向不完整。

### Q4: 目录树接口 `dataTypeId=10`（逻辑模型）返回空？

**原因：** 目录族的类型为"目录族"，其中关联的模型通过 `model/list` 接口（`/tmf-sid-catalogs/{catalogId}/models`）查询，而不是目录树接口。

### Q5: 如何判断接口是否需要分页？

**建议：** 列表接口（`/logical-models` 和 `/data-quality/reports`）默认需要分页；详情接口和血缘接口返回完整数据，无需分页。
