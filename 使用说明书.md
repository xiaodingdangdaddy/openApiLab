# 元数据管理查询接口 - 使用说明书

> 本文档面向接口调用方，提供完整的调用示例、场景说明和常见问题解答。

---

## 一、接口概述

本套 API 提供元数据的统一查询能力，包括：

- **逻辑模型查询**：查询逻辑模型列表、详情、物理模型映射、列定义
- **血缘关系查询**：查询逻辑模型的上下游血缘链路
- **版本历史查询**：查询逻辑模型的版本快照历史和差异比对
- **数据质量查询**：查询数据质量核查结果及评分
- **TMF SID 目录查询**：查询 TMF SID 目录树结构，以及目录下关联的逻辑模型

**Base URL：**

| 环境 | URL |
|------|-----|
| 正式环境 | `https://api.example.com/api/v1` |
| 测试环境 | `https://api-test.example.com/api/v1` |

---

## 二、认证说明

所有接口通过 `Authorization: Bearer <token>` Header 认证。

```bash
curl -k -X GET 'https://api.example.com/api/v1/logical-models?tenantId=example_tenant' \
  -H 'Authorization: Bearer your_token'
```

---

## 三、统一响应格式

所有接口均返回：

```json
{
  "code": "success",
  "msg": "Query success",
  "data": { ... }
}
```

分页结构：

```json
{
  "data": {
    "list": [...],
    "pagination": { "pageNum": 1, "pageSize": 20, "total": 100, "pages": 5 }
  }
}
```

## 四、场景一：查询逻辑模型列表（多维度过滤）

```python
import requests

def get_all_logical_models(tenant_id, token):
    headers = {"Authorization": f"Bearer {token}"}
    all_models = []
    page_num = 1

    while True:
        params = {"tenantId": tenant_id, "pageNum": page_num, "pageSize": 500}
        resp = requests.get(
            "https://api.example.com/api/v1/logical-models",
            headers=headers, params=params
        )
        data = resp.json()
        if data["code"] != "success":
            break
        
        all_models.extend(data["data"]["list"])
        pagination = data["data"]["pagination"]
        if page_num >= pagination["pages"]:
            break
        page_num += 1
    
    return all_models


def get_logical_models_with_filters(
    tenant_id, token,
    theme_id=None,
    catalog_id_path=None,
    owner=None,
    status=None,
    start_create_time=None,
    end_create_time=None,
    page_num=1,
    page_size=20
):
    """
    多维度过滤查询逻辑模型列表

    Args:
        tenant_id: 租户ID（必选）
        token: 认证令牌
        theme_id: 场景ID（模型标签/场景过滤）
        catalog_id_path: 目录ID（递归，该目录及所有子目录下模型）
        owner: 模型责任人（精确匹配）
        status: 模型状态 draft / release
        start_create_time: 创建时间起 YYYY-MM-DD
        end_create_time: 创建时间止 YYYY-MM-DD
        page_num: 页码
        page_size: 每页条数
    """
    headers = {"Authorization": f"Bearer {token}"}
    params = {
        "tenantId": tenant_id,
        "pageNum": page_num,
        "pageSize": page_size,
    }

    if theme_id:
        params["themeId"] = theme_id
    if catalog_id_path:
        params["catalogIdPath"] = catalog_id_path
    if owner:
        params["owner"] = owner
    if status:
        params["status"] = status
    if start_create_time:
        params["startCreateTime"] = start_create_time
    if end_create_time:
        params["endCreateTime"] = end_create_time

    resp = requests.get(
        "https://api.example.com/api/v1/logical-models",
        headers=headers, params=params
    )
    return resp.json()


# ---- 示例用法 ----

# 示例1：按场景（themeId）过滤，查"用户域"标签下所有模型
models = get_logical_models_with_filters(
    tenant_id="example_tenant",
    token="your_token",
    theme_id="T001"
)

# 示例2：按目录递归过滤 + 查 release 状态
models = get_logical_models_with_filters(
    tenant_id="example_tenant",
    token="your_token",
    catalog_id_path="cat_be_001",
    status="release"
)

# 示例3：按责任人过滤
models = get_logical_models_with_filters(
    tenant_id="example_tenant",
    token="your_token",
    owner="admin@example.com"
)
```

## 六、场景三：查询逻辑模型血缘（基础 + 批量模式）

```python
def extract_lineage_edges(lineage_node, edges=None):
    if edges is None:
        edges = []
    
    current_name = lineage_node["tableName"]
    
    for upstream in lineage_node.get("upstream", []):
        edges.append({
            "source": upstream["tableName"],
            "target": current_name,
            "type": "derives_from"
        })
        extract_lineage_edges(upstream, edges)
    
    for downstream in lineage_node.get("downstream", []):
        edges.append({
            "source": current_name,
            "target": downstream["tableName"],
            "type": "influences"
        })
        extract_lineage_edges(downstream, edges)
    
    return edges


def get_lineage(table_name, tenant_id, token, direction="both", depth=3):
    """查询单个模型的上下游血缘链路"""
    headers = {"Authorization": f"Bearer {token}"}
    url = f"https://api.example.com/api/v1/logical-models/{table_name}/lineage"
    resp = requests.get(url, headers=headers, params={
        "tenantId": tenant_id,
        "direction": direction,
        "depth": depth
    })
    return resp.json()


def get_batch_lineage(tenant_id, token, catalog_id_path=None, theme_id=None,
                      direction="both", depth=3):
    """
    批量查询血缘（按目录或场景）

    传入 catalogIdPath 或 themeId，返回该目录下（或该场景下）
    所有模型的各自 lineage 列表。

    Args:
        tenant_id: 租户ID（必选）
        token: 认证令牌
        catalog_id_path: 目录ID（递归，该目录及所有子目录下模型）
        theme_id: 场景ID（该场景下所有模型）
        direction: 血缘方向 upstream/downstream/both
        depth: 递归深度
    """
    headers = {"Authorization": f"Bearer {token}"}
    url = "https://api.example.com/api/v1/logical-models/_batch/lineage"
    params = {"tenantId": tenant_id, "direction": direction, "depth": depth}
    if catalog_id_path:
        params["catalogIdPath"] = catalog_id_path
    if theme_id:
        params["themeId"] = theme_id
    resp = requests.get(url, headers=headers, params=params)
    return resp.json()


# ---- 示例用法 ----

# 示例1：查单个模型的上下游血缘
lineage = get_lineage("dim_user", "example_tenant", "your_token")
edges = extract_lineage_edges(lineage)
for e in edges:
    print(f"{e['source']} --[{e['type']}]--> {e['target']}")

# 示例2：按目录批量查所有模型的 lineage
batch = get_batch_lineage(
    tenant_id="example_tenant",
    token="your_token",
    catalog_id_path="cat_be_001",
    direction="both",
    depth=3
)
for model_lineage in batch.get("data", []):
    print(f"=== {model_lineage['tableName']} ===")

# 示例3：按场景批量查所有模型的 lineage
batch = get_batch_lineage(
    tenant_id="example_tenant",
    token="your_token",
    theme_id="T001"
)
```

## 七、场景四：查询逻辑模型版本历史和差异

```python
def get_version_history(table_name, tenant_id, token, page_num=1, page_size=20):
    """查询版本历史"""
    headers = {"Authorization": f"Bearer {token}"}
    url = f"https://api.example.com/api/v1/logical-models/{table_name}/versions"
    resp = requests.get(url, headers=headers, params={
        "tenantId": tenant_id,
        "pageNum": page_num,
        "pageSize": page_size
    })
    return resp.json()

def diff_versions(table_name, tenant_id, token, from_ver, to_ver):
    """比对两个版本的差异"""
    headers = {"Authorization": f"Bearer {token}"}
    url = f"https://api.example.com/api/v1/logical-models/{table_name}/versions/diff"
    resp = requests.get(url, headers=headers, params={
        "tenantId": tenant_id,
        "from": from_ver,
        "to": to_ver
    })
    data = resp.json()
    
    if data["code"] != "success":
        raise ValueError(f"Diff failed: {data['msg']}")
    
    diff = data["data"]
    summary = diff.get("summary", {})
    print(f"版本 {from_ver} → {to_ver} 变更汇总：")
    print(f"  新增列：{summary.get('columnsAdded', 0)}")
    print(f"  删除列：{summary.get('columnsDeleted', 0)}")
    print(f"  物理模型新增：{summary.get('physicalModelsAdded', 0)}")
    print(f"  物理模型删除：{summary.get('physicalModelsDeleted', 0)}")
    
    for change in diff.get("changedFields", []):
        print(f"  [{change['changeType']}] {change['path']}")
    
    return diff
```

```python
def get_data_quality_results(tenant_id, token, start_date=None, end_date=None):
    """查询 DQ 核查结果列表（基础用法）"""
    headers = {"Authorization": f"Bearer {token}"}
    params = {"tenantId": tenant_id, "pageNum": 1, "pageSize": 500}

    if start_date:
        params["startDataTime"] = start_date
    if end_date:
        params["endDataTime"] = end_date

    all_results = []
    page_num = 1

    while True:
        params["pageNum"] = page_num
        resp = requests.get(
            "https://api.example.com/api/v1/data-quality/results",
            headers=headers, params=params
        )
        data = resp.json()

        if data["code"] != "success":
            break

        all_results.extend(data["data"]["list"])
        pagination = data["data"]["pagination"]
        if page_num >= pagination["pages"]:
            break
        page_num += 1

    return all_results

def get_low_score_results(tenant_id, token, threshold=90):
    """筛选低分核查结果"""
    all_results = get_data_quality_results(tenant_id, token)
    return [r for r in all_results if r.get("score", 0) < threshold]
```

```python
        params["jobInsId"] = job_ins_id
    if catalog_id_path:
        params["catalogIdPath"] = catalog_id_path
    if table_code:
        params["tableCode"] = table_code
    if start_date:

        "https://api.example.com/api/v1/data-quality/results",
```

## 十、场景七：查询 TMF SID 目录结构

```python
def build_catalog_tree(family_catalog_type_id, token, data_type_ids=None):
    """递归构建目录树"""
    if data_type_ids is None:
        data_type_ids = ["1", "2", "3", "4"]
    
    headers = {"Authorization": f"Bearer {token}"}
    catalog_map = {}
    
    for dt in data_type_ids:
        params = {
            "familyCatalogTypeId": family_catalog_type_id,
            "dataTypeId": dt
        }
        resp = requests.get(
            "https://api.example.com/api/v1/tmf-sid-catalogs",
            headers=headers, params=params
        )
        data = resp.json()
        if data["code"] != "success":
            continue
        
        for node in data["data"]:
            catalog_map[node["catalogId"]] = node
            catalog_map[node["catalogId"]]["children"] = node.get("children", [])
    
    # 构建父子关系
    roots = []
    for node in catalog_map.values():
        parent_id = node.get("parentCatalogId")
        if parent_id and parent_id in catalog_map:
            catalog_map[parent_id]["children"].append(node)
        else:
            roots.append(node)
    
    return roots
```

## 十一、版本变更规则说明

### 10.1 触发版本 +1 的变化

| 变化类型 | 说明 |
|---------|------|
| 逻辑列新增 | `logicalColumns` 中出现新的 columnName |
| 关联物理模型新增 | `relatedPhysicalModels` 中出现新的物理模型 |
| 关联物理模型删除 | `relatedPhysicalModels` 中某个物理模型消失 |
| 物理表列新增 | `physicalColumns` 中出现新的列 |

### 10.2 系统不允许的操作（不涉及版本变化）

| 操作 | 说明 |
|------|------|
| 逻辑列删除 | 系统不允许 |
| 物理列删除 | 系统不允许 |
| 列重命名 | 系统不允许 |
| 列数据类型变化 | 系统不允许 |
| 列映射表达式变化 | 系统不允许 |
| status 状态发布变化 | 系统不允许 |
| openFlag 开放标识变化 | 系统不允许 |
| 别名/描述修改 | 静默保存 |

### 10.3 数据质量

不做版本变化追踪。每次 DQ 核查运行生成唯一的 `jobInsId`，即一个核查结果快照。直接使用 `score` 字段作为质量评分指标。

## 十三、常见问题

**Q1: 返回 `code=fail`，`msg=Invalid param`？**

检查必选参数是否传入，`dataTypeId` 是否在 `["1","2","3","4","10"]` 范围内（字符串类型）。

**Q2: 逻辑模型详情里 `relatedPhysicalModels` 是空数组？**

该模型尚未关联物理表。

**Q3: 血缘接口 `forwardLineage` 和 `backwardImpact` 都是空？**

该模型尚未录入血缘关系。

**Q4: 目录树 `dataTypeId=10`（逻辑模型）返回空？**

逻辑模型不通过目录树接口查询，通过 `/tmf-sid-catalogs/{catalogId}/models` 查询。

**Q5: 版本历史 `versions` 是空的？**

说明该模型尚未发生列结构或物理模型变化，版本快照表为空。
